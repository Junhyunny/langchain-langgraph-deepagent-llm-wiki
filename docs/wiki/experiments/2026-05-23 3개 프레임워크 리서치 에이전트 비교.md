# 실험: 3개 프레임워크 리서치 에이전트 비교

## 목표

동일한 리서치 에이전트 작업을 LangChain, LangGraph, Deep Agents로 각각 구현하고, 오케스트레이션 방식, 에이전트 런타임 구성 요소(가드레일, 핸드오프, 메모리), 코드 복잡도를 비교한다.

## 설정

**작업:** "Python의 GIL(Global Interpreter Lock)이 무엇인지 설명하고, 이를 우회하는 방법을 정리해라."

**공통 도구:** 웹 검색 도구 1개 (또는 mock search tool)

**비교 기준:**
1. 코드 양 (Lines of Code)
2. 오케스트레이션 방식 (명시적 그래프 vs 암묵적 루프)
3. 메모리/상태 관리 방식
4. 스트리밍 지원 방식
5. 가드레일 추가 용이성

## 실험 코드

### 1. LangChain (`langchain.agents.create_agent`)

```python
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langchain.agents import tool

@tool
def search(query: str) -> str:
    """Search the web for information."""
    # mock search
    return f"Search results for: {query}"

model = ChatOpenAI(model="gpt-4o-mini")

agent = create_agent(
    model=model,
    tools=[search],
    instructions="You are a research assistant. Search for information and provide comprehensive answers."
)

# 단순 실행
result = agent.invoke({"messages": [{"role": "user", "content": "Python GIL에 대해 설명하고 우회 방법을 정리해라."}]})
print(result["messages"][-1].content)
```

**특징:**
- `create_agent`가 내부적으로 LangGraph StateGraph를 생성
- 오케스트레이션은 프레임워크가 자동 처리
- 가드레일 추가 = 별도 구현 필요 (표준 API 없음)

### 2. LangGraph (`StateGraph` 직접 구성)

```python
from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, MessagesState, END
from langgraph.prebuilt import ToolNode
from langchain.agents import tool

@tool
def search(query: str) -> str:
    """Search the web for information."""
    return f"Search results for: {query}"

model = ChatOpenAI(model="gpt-4o-mini").bind_tools([search])

def agent_node(state: MessagesState):
    return {"messages": [model.invoke(state["messages"])]}

def should_continue(state: MessagesState):
    last_message = state["messages"][-1]
    if last_message.tool_calls:
        return "tools"
    return END

builder = StateGraph(MessagesState)
builder.add_node("agent", agent_node)
builder.add_node("tools", ToolNode([search]))
builder.set_entry_point("agent")
builder.add_conditional_edges("agent", should_continue)
builder.add_edge("tools", "agent")

graph = builder.compile()

result = graph.invoke({
    "messages": [{"role": "user", "content": "Python GIL에 대해 설명하고 우회 방법을 정리해라."}]
})
print(result["messages"][-1].content)
```

**특징:**
- 오케스트레이션을 명시적으로 정의 (그래프 구조)
- 가드레일 = 별도 노드로 추가 가능
- 메모리/체크포인팅 = `compile(checkpointer=...)` 추가

### 3. Deep Agents (`create_deep_agent`)

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from langchain.agents import tool

@tool
def search(query: str) -> str:
    """Search the web for information."""
    return f"Search results for: {query}"

model = ChatOpenAI(model="gpt-4o-mini")

agent = create_deep_agent(
    model=model,
    tools=[search],
    instructions="You are a research assistant. Search for information and provide comprehensive answers."
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "Python GIL에 대해 설명하고 우회 방법을 정리해라."}]
})
print(result["messages"][-1].content)
```

**특징:**
- `create_agent`에 위임하지만 Middleware 계층 추가
- HarnessProfile로 모델별 설정 자동화
- 메모리 = `MemoryMiddleware` + `CompositeBackend` 기본 제공

## 비교 표 (예상)

| 기준 | LangChain | LangGraph | Deep Agents |
|------|-----------|-----------|-------------|
| 최소 코드 (LOC) | ~15 | ~30 | ~15 |
| 오케스트레이션 | 암묵적 | 명시적 그래프 | 암묵적 (미들웨어) |

| 핸드오프 | `Command(goto=)` | `Command(goto=)` | subagent tool |
| 스트리밍 | `astream_events` | `astream_events` | `astream_events` |
