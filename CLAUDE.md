# 하네스 운영 규칙 (최우선)

이 저장소는 **학습용 LLM 위키**다.
LangChain · LangGraph · Deep Agents를 소스 코드와 공식 문서 수준까지
이해하는 것이 목적이다. 이 저장소 자체가 산출물이지, 외부에 기여하는
작업 공간이 아니다.

## 1. 저장소 쓰기 작업 금지 (GitHub)

이 저장소를 포함한 **어떤 저장소에도** 다음을 수행하지 않는다.

- Pull Request 생성 / 수정 / 머지
- 이슈 생성 / 수정 / 코멘트 작성
- 브랜치 생성, fork, 파일 commit/push (GitHub API 경유)
- 리뷰 작성, 코멘트 응답, 리뷰 스레드 해결

GitHub은 **읽기 전용**으로만 쓴다 — 소스 코드 읽기, 이슈/PR **분석**,
검색. 이것은 위키의 "이슈 분석" 페이지와 "코드 리딩" 워크플로를 위한
것이다. 무언가를 *만들거나 올리려는* 충동이 들면 멈추고, 대신 그 내용을
위키 페이지(분석/요약)로 적는다.

> 명시적으로 사용자가 "PR 올려줘" 같은 요청을 한 경우에만 예외다.
> 그런 요청이 없으면 절대 쓰기 도구를 호출하지 않는다.

## 2. 학습 무관 MCP 도구 호출 금지

Notion, Figma, Google Drive 등 **이 위키의 학습 워크플로와 무관한
MCP 서버 도구는 호출하지 않는다.** 도구 이름(또는 서버 ID)이 바뀌어도
이 규칙은 의도로서 적용된다. 학습에 쓰는 도구는 다음뿐이다.

- 로컬 파일 도구 (Read / Edit / Write / Grep / Glob)
- `WebFetch` / `WebSearch` (공식 문서·소스 수집용)
- GitHub MCP의 **읽기** 도구
- 이 저장소의 skills (`fetch-raw-doc`, `update-wiki`, `wiki-ask`, `grill-me`)

## 3. 학습 목표 게이트 (자율 수집 금지)

리소스 수집(raw fetch, 이슈 읽기, 소스 다운로드)은 **명시적 학습
질문/주제가 있을 때만** 시작한다. 주제가 없으면 수집하지 말고 먼저
물어본다: *"오늘 어떤 주제를 학습할까요?"*

raw 자료를 추가하기 전 AGENTS.md의 자문 질문을 통과해야 한다.

- 왜 이 소스가 필요한가?
- 어떤 위키 페이지가 이를 사용할 것인가?
- 안정적인 URL 또는 commit SHA가 있는가?

"혹시 몰라서" 또는 "유휴 상태를 메우려고" 하는 수집은 금지다.

## 4. "다음 작업" 제안 범위 (화이트리스트)

먼저 다음 작업을 제안할 때는 **위키 워크플로 내부 작업**으로만 한정한다.

- 기존 위키 페이지 보강 / 검증 라벨 갱신
- 미흡한 `source_summary` 작성
- `_open_questions.md`의 열린 질문 해소
- 사용자가 고른 학습 주제에 대한 raw 수집 → 요약 → 위키 반영
- 소스 코드 읽기 세션 결과를 flow/code-map 페이지로 정리

이 범위 밖의 작업(외부 기여, 무관한 리소스 수집, 대규모 리팩터링)은
사용자가 명시적으로 요청하기 전에는 제안하지 않는다.
