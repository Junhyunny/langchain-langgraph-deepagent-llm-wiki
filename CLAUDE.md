# 하네스 운영 규칙 (최우선)

이 저장소는 **학습용 LLM 위키**다.
LangChain · LangGraph · Deep Agents를 소스 코드와 공식 문서 수준까지
이해하는 것이 목적이다.

**최종 목표:** 오픈소스 PR 기여가 가능한 수준의 지식을 갖춘다.
**학습의 중심축:** `docs/wiki/_book_roadmap.md` — 100개 섹션, 각 섹션이
블로그 글 1편이자 학습 사이클 1회다. AI는 사용자가 요청한 섹션을
준비하는 역할만 한다. 요청 없이 먼저 제안하거나 수집하지 않는다.

## 1. 저장소 쓰기 작업 금지 (GitHub)

이 저장소를 포함한 **어떤 저장소에도** 다음을 수행하지 않는다.

- Pull Request 생성 / 수정 / 머지
- 이슈 생성 / 수정 / 코멘트 작성
- 브랜치 생성, fork, 파일 commit/push (GitHub API 경유)
- 리뷰 작성, 코멘트 응답, 리뷰 스레드 해결

GitHub은 **읽기 전용**으로만 쓴다 — 소스 코드 읽기, 이슈/PR **분석**,
검색. 이것은 위키의 "이슈 분석" 페이지와 "코드 리딩" 워크플로를 위한
것이다. 무언가를 *만들거나 올리려는* 충동이 들면 멈추고, 대신 그 내용을
위키 페이지(분석/요약)로 적는다.

> 명시적으로 사용자가 "PR 올려줘" 같은 요청을 한 경우에만 예외다.
> 그런 요청이 없으면 절대 쓰기 도구를 호출하지 않는다.

## 2. 학습 무관 MCP 도구 호출 금지

Notion, Figma, Google Drive 등 **이 위키의 학습 워크플로와 무관한
MCP 서버 도구는 호출하지 않는다.** 도구 이름(또는 서버 ID)이 바뀌어도
이 규칙은 의도로서 적용된다. 학습에 쓰는 도구는 다음뿐이다.

- 로컬 파일 도구 (Read / Edit / Write / Grep / Glob)
- `WebFetch` / `WebSearch` (공식 문서·소스 수집용)
- GitHub MCP의 **읽기** 도구
- 이 저장소의 skills (`fetch-raw-doc`, `update-wiki`, `wiki-ask`, `grill-me`)

## 3. 학습 목표 게이트 (자율 수집·제안 금지)

모든 작업은 **사용자의 명시적 요청**이 있을 때만 시작한다.
사용자가 요청하지 않으면 어떤 것도 먼저 제안하거나 수집하지 않는다.

작업 범위는 `_book_roadmap.md`의 미완료(`[ ]`) 섹션으로 한정한다.
로드맵에 없는 주제의 raw 수집, 위키 생성, 요약 작성은 금지다.

raw 자료를 추가하기 전 반드시 자문한다.

- 이 소스는 `_book_roadmap.md`의 어느 섹션을 위한 것인가?
- 어떤 위키 페이지가 이를 사용할 것인가?
- 안정적인 URL 또는 commit SHA가 있는가?

"혹시 몰라서" 또는 "유휴 상태를 메우려고" 하는 수집은 금지다.

## 4. 허용된 작업 범위 (화이트리스트)

사용자가 특정 `_book_roadmap.md` 섹션을 요청했을 때만 아래를 수행한다.

- 해당 섹션의 관련 위키 페이지 보강 및 검증 라벨 갱신
- 관련 소스의 source_summary 작성
- 관련 예제 코드 또는 실험 결과 연결
- 소스 코드 읽기 결과를 flow/code-map 페이지로 정리

`_open_questions.md`는 리서치 리소스 수집 전용이다. 책 섹션 진행의
드라이버가 아니다.

아래는 사용자가 명시적으로 요청하지 않는 한 절대 하지 않는다.

- `docs/wiki/prs/` 같은 PR 준비 문서 생성 (PR 작성은 사용자가 직접 한다)
- `_book_roadmap.md`에 없는 섹션 자율 추가
- 외부 저장소 기여, 대규모 리팩터링, 무관한 리소스 수집
