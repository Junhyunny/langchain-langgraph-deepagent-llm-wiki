# sync-coverage

`docs/wiki/_book_roadmap.md`의 각 섹션 제목 끝 준비도 기호를 실제 파일 상태에 맞게 갱신한다.

---

## 판정 규칙

각 섹션의 기호는 해당 섹션에 연결된 위키 페이지(들)의 `status` frontmatter 값으로 결정한다.

| 기호 | 조건 |
|------|------|
| ✅ | 연결된 모든 주요 위키 페이지가 `status: verified` |
| 🟠 | 연결된 위키 페이지가 `status: partial`, 또는 일부는 verified · 일부는 draft인 혼합 상태 |
| 🟡 | 연결된 위키 페이지가 존재하고 가장 높은 status가 `draft` |
| ❌ | 연결된 위키 페이지가 없음 |

**복수 페이지 규칙:**
- 모두 verified → ✅
- 하나라도 partial 또는 verified·draft 혼합 → 🟠
- 모두 draft → 🟡
- 하나도 없음 → ❌

---

## 실행 절차

### 1단계 — 위키 상태 수집

다음 명령으로 모든 위키 페이지의 `status` 값을 한 번에 수집한다.

```bash
grep -r "^status:" docs/wiki --include="*.md" -l | sort | while read f; do
  s=$(grep "^status:" "$f" | head -1)
  echo "$f: $s"
done
```

결과를 내부적으로 `파일 경로 → status` 맵으로 보유한다.

### 2단계 — 로드맵 읽기

`docs/wiki/_book_roadmap.md`를 읽고, 아래 두 가지를 섹션별로 파악한다.

1. 섹션 행: `- [ ] **X.Y** 제목 — 부제목 🟡` 형식의 행
2. 연결된 위키 페이지: 바로 다음 줄의 `  - 위키: [[PageName]]` 패턴

### 3단계 — 위키 페이지 파일 경로 해석

`[[PageName]]`을 실제 파일 경로로 매핑한다. 검색 순서:

1. `docs/wiki/**/*PageName*.md` 패턴으로 글로브 검색
2. 파일명이 `PageName`과 정확히 일치하는 파일 우선
3. 없으면 ❌ 처리

`위키:` 행이 없는 섹션은 섹션 번호와 키워드를 기반으로 관련 위키 페이지를 탐색한다.
탐색 결과가 없으면 ❌.

### 4단계 — 기호 결정

위 판정 규칙을 적용해 각 섹션의 새 기호를 결정한다.

### 5단계 — `_book_roadmap.md` 수정

변경이 필요한 섹션만 수정한다.

섹션 행의 기존 기호(✅/🟠/🟡/❌)를 새 기호로 교체한다.

```
기존: - [ ] **4.1** Runnable 인터페이스란? — ... 🟡
교체: - [ ] **4.1** Runnable 인터페이스란? — ... ✅
```

기호가 없는 섹션 행이 있으면 제목 끝에 추가한다.

**건드리지 않는 것:**
- `  - 위키: [[...]]` 보조 행
- `[x]` 완료 체크 여부
- 범례 섹션 (`## 위키 준비도 범례`)
- 진행 현황 표

### 6단계 — 변경 요약 보고

변경된 섹션 수와 방향(예: 🟡→✅, ❌→🟡)을 요약 리포트로 출력한다.

예시:
```
coverage 갱신 완료 (3개 변경)
- 4.1 Runnable: 🟡 → ✅
- 5.1 PromptTemplate: ❌ → 🟡
- 17.1 create_deep_agent: 🟡 → 🟠
```

---

## 주의

- 이 스킬은 기호만 갱신하며 위키 페이지 내용이나 다른 로드맵 내용은 수정하지 않는다.
- `[x]` 완료 체크된 섹션도 기호는 그대로 갱신한다 (블로그 발행 ≠ 위키 준비도).
- 변경 전후 기호가 동일한 섹션은 건드리지 않는다.
- 스킬 실행 후 커밋 여부는 사용자에게 물어본다.
